explanation
------------

This project is a backend e-commerce REST API developed using Spring Boot with a strong focus on security, scalability, and clean role
 separation using Spring Security and JWT. The application is divided into three modules: Admin, Merchant, and Customer. 
 The Admin account is created using CommandLineRunner and can log in, update passwords, manage merchants and customers, block or unblock users,
  and approve products added by merchants, ensuring overall platform integrity. Merchants can log in, update their credentials, and perform full CRUD 
  operations on products, which become visible to customers only after admin approval to maintain quality control. Customers can register with email verification
   using OTP stored in Redis and sent asynchronously, view only approved products, manage cart operations, place orders, make payments through Razorpay integration,
    and view order history. The application uses stateless JWT-based authentication with role-based access enforced through @PreAuthorize annotations. Currently,
     the project is backend-only, and I am learning React to build the frontend and make it a complete end-to-end e-commerce application.
